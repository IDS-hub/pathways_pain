AuthProviders [1.0]

Third party authorization providers

create

Sign in given user by thrid-party provider (e.g. facebook, google etc.)

URL

POST api/v1/auth_providers

Required Parameters

  • login String

    user login. Usually is email
  • provider_info Hash

    provider information data

Optional Parameters

  • first_name String

    first name of user
  • last_name String

    last name of the given user

Examples

Example for invalid response Request 
{
 "first_name": "Test",
 "last_name": "Fb user",
 "login": "test_login@gmail.com",
 "provider_info": {
   "provider_name": "facebook",
   "provider_token": "EAAKdIoryhDxglwSqLCbmc7uxuqEjlM0qYUXPnxPUOlkZCKRUfjZCX4aIZD"
 }
}
